PROIEL samples

Representative sentences from each diachronic period of the AthDGC corpus, with full PROIEL XML 2.0 annotation and dependency-tree visualization

Every sample on this page is real, annotated PROIEL XML 2.0 output from the AthDGC pipeline. For each sample you see (i) the source citation, (ii) the dependency tree, (iii) the per-verb argument-structure frame, and (iv) the raw PROIEL XML.

No CoNLL-U. No UD export. PROIEL XML 2.0 only.

tree key: indentation = depth · ▸ = head · italic label = PROIEL relation (sub, obj, obl, adv, atr, pred, aux, comp, xobj, nonsub, etc.)

Four visualisations of Iliad 1.1

μῆνιν ἄειδε θεὰ Πηληϊάδεω Ἀχιλῆος

The same PROIEL XML 2.0 annotation can be rendered in several conventions. Below are four ways AthDGC presents a single sentence: the project’s default indented tree, a Grew/Arborator arc diagram, a Perseus-style descending tree, and a Lavidasized typographic layout designed for the AthDGC house style.

1. AthDGC default — indented tree

▸ ἄειδε        pred · imp.act.2sg
  ├ μῆνιν       obj  · acc.sg.f
  │   └ Ἀχιλῆος    atr  · gen.sg.m
  │       └ Πηληϊάδεω  atr  · gen.sg.m
  └ θεά         voc  · voc.sg.f

2. Grew / Arborator style — arc diagram

Tokens on a baseline; arcs above; each arc labelled with the PROIEL relation; arrow-heads from head to dependent.

3. Perseus style — descending tree

Head at the top, dependents hanging below. Edges carry the PROIEL relation.

4. Lavidasized typographic — new AthDGC house style

Three-column austere typography. Token form in serif on the left (indented by dependency depth), italic relation label in the centre, PROIEL morphology in monospace on the right. Thin rules between rows. No boxes. No colours.

How to read these samples

Each sentence is annotated under the PROIEL XML 2.0 schema with the relation inventory established by Haug & Jøhndal (2008) and extended by Eckhoff et al. (2018):

  • pred — root predicate
  • sub / obj — core arguments
  • obl — oblique (prepositional / instrumental / locative / ablative)
  • atr — attributive (adnominal modifier)
  • adv — adverbial modifier
  • voc — vocative
  • aux — auxiliary (article, copula component)
  • xcomp / comp — open / closed complement clauses
  • xobj / nonsub — raised arguments
  • pred-nom — nominal predicate (with copula)

Every annotated token additionally carries full morphology (the 10-character PROIEL morphological tag) and lemma. The per-verb argument-structure frame is derived from these PROIEL relations directly — no UD remapping, no CoNLL-U export step.
